Java 보안

Java 언어는 처음부터 보안을 염두에 두고 설계되었습니다. 이는 다른 많은 기존 시스템 및 플랫폼에 비해 큰 이점을 제공합니다.

Java 보안 아키텍처는 보안 전문가에 의해 설계되었으며, 플랫폼이 시작된 이래로 다른 많은 보안 전문가에 의해 연구되고 검증되었습니다.

Java 보안 아키텍처의 근본은 바이트 코드가 표현할 수 있는 내용에 강한 제약을 두는 것입니다. 예를 들어 메모리를 직접 처리할 수 있으며, 이는 GC(Garbage Collector)를 통해 간접적으로 이루어집니다. 이는 C/C++ 등 하드웨어와 가깝게 설계된 기존 언어를 괴롭혔던 보안 문제이기도 합니다. 또한 JVM 클래스 로더는 바이트 코드 검증 과정을 통해 신뢰할 수 있는 클래스만을 읽어냅니다.

물론 그럼에도 불구하고 그 어떤 시스템도 100% 보안을 보장할 수는 없으며 Java 도 예외는 아닙니다. 이론적으로 견고하게 설계되었다고 할 지라도 보안 아키텍처의 구현은 또 다른 문제입니다.

실제로 특정 Java 구현에서 보안 취약점이 발견되어 개선되었던 지난 역사가 있습니다. 특히 Java 8 의 출시는 상당한 노력이 필요한 다수의 보안 문제가 발견되어 부분적으로 지연되었습니다. 앞으로도 계속 JVM 구현에서 보안 취약점이 발견될 가능성은 존재합니다.

그러나 최근 Java 의 보안 취약점의 대부분은 데스크탑 기술 분야와 밀접하게 연관되어 있었다는 점도 주목할 가치가 있습니다. 현실적으로 서버 사이드 코딩의 경우, Java 는 현재도 굉장히 안전한 범용 플랫폼으로 평가받고 있습니다.
