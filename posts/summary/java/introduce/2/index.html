Java, JVM 의 간략한 역사

  • Java 1.0 (1996)

Java 의 최초 공개 버전으로, 8개의 패키지로 구성된 212개의 클래스만 포함

  • Java 1.1 (1997)

Inner Class, Reflection API 도입 등

  • Java 1.2 (1998)

Set, Map, List 등 Java Collection API 도입 등

새로운 기능으로 인해 Sun Microsystems 는 Java 2 플랫폼으로 상표명을 변경

(Java 2 플랫폼은 단순한 상표명일 뿐 실제 출시 버전 번호와는 다름)

  • Java 1.3 (2000)

주로 버그 수정, 안정성 및 성능 개선에 초점을 맞춘 업데이트

HotSpot JVM 도입 등

  • Java 1.4 (2002)

더 높은 성능을 제공하는 저수준 I/O API 도입 등

(텍스트 처리를 위한 정규식, XML 및 XSLT 라이브러리, SSL 지원, 로깅 API 및 암호화 지원 등)

  • Java 5 (2004)

Generic, Enum, Annotation, Varargs Method, Autoboxing, 새로운 for 루프 문법을 포함하여 핵심 언어 자체에 많은 변경 사항이 적용

출시 버전 체계가 크게 변화하였으며, 166개 패키지로 구성된 3,562개의 클래스와 인터페이스가 포함

동시성 프로그래밍을 위한 유틸리티, 원격 관리 프레임워크 도입 등

  • Java 6 (2006)

주로 유지 관리 및 성능을 위한 출시

컴파일러 API 도입, 주석의 용도와 범위를 확장, 스크립팅 언어와 상호 운용될 수 있도록 해주는 바인딩 제공 등

  • Java 7 (2011)

Oracle 이 관리하는 첫 번째 Java 출시 버전

try-with-resources, NIO.2 API, Method Handles API 도입 등

  • Java 8 (2014)

람다 표현식 도입으로 함수형 프로그래밍 패러다임이 새롭게 대두됨

Nashorn JVM 에서 실행되는 JavaScript 구현, 새로운 날짜 및 시간 지원, Java 프로필 도입 등
