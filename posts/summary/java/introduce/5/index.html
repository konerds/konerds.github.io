Java 와 다른 언어의 비교


Java vs C

  • Java 는 객체 지향적인 반면, C 는 절차 지향적입니다.
  • Java 는 바이트 코드 파일로 이식이 가능한 반면, C 는 재컴파일이 필요합니다.
  • Java 는 런타임 도중 바이트 코드를 변경하는 기능을 가지고 있습니다.
  • Java 에는 포인터 및 포인터 산술이 존재하지 않습니다.
  • Java 는 가비지 컬렉션을 통해 자동으로 메모리를 관리할 수 있습니다.
  • Java 에는 구조체가 존재하지 않으며, 로우 레벨에서 메모리를 배치하는 기능이 없습니다.
  • Java 에는 전처리기가 없습니다.

Java vs C++

  • Java 는 C++ 에 비해 단순화된 객체 모델을 가지고 있습니다.
  • Java 의 디스패치는 기본적으로 가상입니다.
  • Java 는 항상 값으로 전달됩니다. (객체 참조 역시 포인터 개념과는 다름)
  • Java 는 완전한 다중 상속을 지원하지 않습니다.
  • Java 의 제네릭은 C++ 템플릿보다 덜 효율적일 수 있으나, 보다 안전합니다.
  • Java 에는 연산자 오버로딩이 존재하지 않습니다.

Java vs PHP

  • Java 는 정적으로 타입이 지정되나, PHP 는 동적으로 타입이 지정됩니다.
  • Java 는 PHP 에 비해 범용적으로 사용되는 언어입니다.
  • Java 는 다중 스레드로 동작하나, PHP 는 그렇지 않습니다.

Java vs Javascript

  • Java 는 정적으로 타입이 지정되나, Javascript 는 동적으로 타입이 지정됩니다.
  • Java 는 클래스 기반 객체를 사용하나, Javascript 는 프로토타입을 기반으로 합니다.
  • Java 에는 네임스페이스 개념이 존재하나, Javascript 에는 존재하지 않습니다.
  • Java 는 다중 스레드로 동작하나, Javascript 는 그렇지 않습니다.
