Java 문법

이 섹션은 Java 문법에 대해 간결하되 포괄적으로 알아봅니다.

가장 낮은 수준의 Java 문법에서 시작하여 점점 더 높은 수준의 구조를 다루면서 Java 문법을 알아봅니다.

  • Java 프로그램을 작성하는 데 사용되는 문자와 해당 문자의 인코딩
  • Java 프로그램을 구성하는 리터럴 값, 식별자 및 기타 토큰
  • Java 로 다룰 수 있는 데이터 타입
  • Java 에서 개별 토큰을 더 큰 표현식으로 그룹화하는 데 사용되는 연산자
  • 표현식 및 기타 명령문을 그룹화하여 Java 코드의 논리적 청크를 형성하는 명령문
  • 다른 Java 코드에서 호출할 수 있는 명명된 메서드.
  • 메서드와 필드의 모음이자, Java 객체 지향 프로그래밍의 핵심 요소인 클래스
  • 관련 클래스의 모음인 패키지
  • 하나 이상의 패키지에서 끌어올 수 있는 하나 이상의 상호 작용 클래스로 구성된 Java 프로그램

대부분의 프로그래밍 언어의 구문은 복잡하며 Java 도 예외는 아닙니다. 일반적으로 아직 논의되지 않은 다른 요소를 참조하지 않고 언어의 모든 요소를 문서화하는 것은 불가능합니다.

예를 들어, 객체를 참조하지 않고 Java 가 지원하는 연산자와 명령문을 의미 있는 방식으로 설명하는 것은 실제로 불가능합니다. 그러나 언어의 연산자와 명령문을 참조하지 않고 객체를 철저하게 문서화하는 것도 불가능합니다.

따라서 Java 또는 모든 언어를 학습하는 과정은 반복적인 과정입니다.


#1 - 하향식으로 Java 프로그램 개요 살펴보기
December 5, 2023

하향식으로 Java 프로그램 개요 살펴보기 Java 문법에 대한 상향식 탐색을 시작하기 전에 잠시 하향식으로 Java 프로그램을 살펴보겠습니다. Java 프로그램은 하나 이상의 Java 소스 코드 파일 또는 컴파일 단위로 구성됩니다. 이 섹션의 마지막 부분에서는 Java 파일의 구조를 설명하고 Java 프로그램을 컴파일하고 실행하는 방법을 설명합니다. 각 컴파일 단위는 선택적 package 선언으로 시작하고 그 뒤에 0개 이상의 import 선언이 옵니다. 이러한 선언은 컴파일 단위가 이름을 정의할 네임스페이스와 컴파일 단위가 이름을 가져오는 네임스페이스를 지정합니다.
